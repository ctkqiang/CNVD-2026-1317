## 漏洞概述

### 什么是 CNVD-2026-13173

CNVD-2026-13173 是一个影响 QuestDB 数据库的未授权访问漏洞。QuestDB 是一个开源的时间序列数据库，以其极快的导入速度和动态、低延迟的 SQL 查询能力而闻名。

### 漏洞详情

- **漏洞类型**：未授权访问漏洞
- **危害级别**：中 (AV:N/AC:L/Au:N/C:P/I:N/A:N)
- **影响产品**：QuestDB
- **公开日期**：2026-02-14
- **报送日期**：2026-02-26
- **收录日期**：2026-03-10

### 漏洞描述

QuestDB 存在未授权访问漏洞，攻击者可利用该漏洞获取敏感信息。具体来说，QuestDB 的认证配置失效导致未授权用户能够访问系统中的敏感数据。

### 解决方案

- **临时解决方案**：在 QuestDB 前端部署反向代理（如 Nginx）并启用 HTTP Basic 认证，作为额外的访问控制层
- **正式解决方案**：厂商尚未提供漏洞修复方案，请关注厂商主页更新：https://questdb.com/

## 项目结构

本项目是一个用于测试 CNVD-2026-13173 漏洞的工具，主要包含以下文件：

```
CNVD-2026-13173/
├── inject.cc          # 主测试工具（C++版本）
├── README.md          # 项目说明文档
└── hosts.txt          # 目标主机列表示例（可选）
```

### 文件说明

- **inject.cc**：主要测试工具，使用 C++ 编写，集成 libcurl 库进行 HTTP 请求处理，支持批量测试多个主机
- **README.md**：本说明文档，包含漏洞信息、使用方法和教学内容
- **hosts.txt**：目标主机列表文件示例，每行一个主机地址

## 工具功能

1. **多主机批量测试**：支持从文件读取多个目标主机地址
2. **循环执行模式**：支持有限次数循环和无限循环两种模式
3. **随机 SQL 生成**：自动生成随机的表名和列定义进行注入测试
4. **实时进度显示**：显示每次注入的详细信息和 HTTP 响应状态
5. **中文语义化代码**：使用中文宏定义提升代码可读性

## 编译教程

### 步骤 1：安装依赖

**Ubuntu/Debian 系统**：
```bash
sudo apt update
sudo apt install g++ libcurl4-openssl-dev
```

**CentOS/RHEL 系统**：
```bash
sudo yum install gcc-c++ curl-devel
```

**macOS 系统**：
```bash
brew install gcc curl
```

### 步骤 2：编译工具

在项目目录中执行以下命令：

```bash
g++ -o inject inject.cc -lcurl -std=c++11 -O2 -Wall -Wextra
```

**编译选项说明**：
- `-o inject`：指定输出可执行文件名为 inject
- `-lcurl`：链接 libcurl 库
- `-std=c++11`：使用 C++11 标准
- `-O2`：优化级别 2，平衡性能与代码大小
- `-Wall -Wextra`：开启额外警告信息，提升代码质量

### 步骤 3：验证编译结果

编译成功后，会在当前目录生成一个名为 `inject` 的可执行文件。可以通过以下命令查看文件是否生成：

```bash
ls -la inject
```

## 使用教程

### 基本语法

```bash
./inject -f <主机文件> -l <循环次数|inf>
```

### 参数说明

1. **-f, --file <主机文件>**：指定包含目标主机列表的文件路径
   - 文件格式：每行一个主机地址，支持 IP、域名或完整 URL
   - 支持注释：以 # 开头的行将被忽略
   - 示例文件内容：
     ```
     # 测试目标列表
     192.168.1.100
     localhost
     http://example.com:9000
     ```

2. **-l, --loop <循环次数|inf>**：指定循环执行次数
   - 正整数：执行指定次数后自动停止
   - inf 或 0：无限循环模式，需手动停止（Ctrl+C）
   - 默认值：1 次（如果不指定 -l 参数）

### 使用示例

#### 示例 1：对单主机执行 1 次测试

```bash
./inject -f hosts.txt -l 1
```

#### 示例 2：对多主机执行 5 次循环测试

```bash
./inject -f target_hosts.txt -l 5
```

#### 示例 3：无限循环测试模式（用于持续监控）

```bash
./inject -f hosts.txt -l inf
```

#### 示例 4：使用默认 localhost 目标进行测试

```bash
./inject -l 3
```

### 输出说明

程序运行时将显示以下信息：

1. **目标主机列表**：显示所有要测试的主机地址
2. **循环模式配置**：显示当前的循环模式和次数
3. **注入详情**：每次注入的详细信息，格式为 `[时间] 正在注入表: <表名> @ <主机> | 状态码: <HTTP状态>`
4. **循环进度**：显示当前循环次数和总次数

### 结果解读

- **HTTP 状态码 200**：请求成功，目标可能存在漏洞
- **HTTP 状态码 401/403**：访问被拒绝，目标可能已修复漏洞
- **HTTP 状态码 404**：请求的路径不存在
- **其他状态码**：需要根据具体情况分析

## 故障排除

### 常见问题及解决方法

1. **编译失败**：
   - 检查是否安装了 libcurl 开发库
   - 确保编译器支持 C++11 标准
   - 检查编译命令是否正确

2. **运行时错误**：
   - **curl_easy_init 失败**：检查 libcurl 库是否正确安装
   - **无法连接到主机**：检查网络连接和目标主机是否可达
   - **权限错误**：确保对主机文件有读取权限

3. **测试结果异常**：
   - 检查目标主机是否运行 QuestDB
   - 确认目标主机端口是否正确（默认 9000）
   - 验证网络防火墙是否允许访问
